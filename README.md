# StackBar

Windows タスクバー グループランチャー

---

## 概要

純正タスクバーのアイコンが増えすぎる問題を解決するアプリ。  
複数のアプリをグループにまとめ、スマホのフォルダのようにポップアップで管理できる。

---

## 仕組み

1. StackBar の設定アプリでグループを作成し、純正タスクバーにピン留めする
2. グループアイコンをクリックすると、アイコンの真上にアプリ一覧がポップアップ表示される
3. アプリをクリックして起動する

---

## アプリ構成

| コンポーネント | 役割 |
|---|---|
| ポップアップ本体 | タスクトレイに常駐。グループアイコンをクリックするとアプリ一覧を表示 |
| 設定・管理アプリ | タスクトレイから開く。グループの作成・編集・削除、デザイン設定、言語設定を管理 |

---

## 機能一覧

### グループアイコン（純正タスクバー上）

- 純正タスクバーと同じサイズで表示
- グループ内アプリのアイコンを最大4つ小さく表示（2×2グリッド）
- アプリが1個のときは小さく中央表示
- 4つを超えた分は最初の4つのみ表示
- ホバーでグループ名を表示（純正タスクバーと同じ挙動）

### ポップアップ

- グループアイコンをクリックで開く
- アプリをクリックして起動、または他の場所をクリックで閉じる
- アイコンサイズはWindowsタスクバーと同じサイズ
- ホバーでアプリ名をアイコン上に表示
- グループ名をポップアップ上部に表示
- ドラッグ&ドロップでアプリの並び替え可能

### アプリの追加・削除・移動

- 純正タスクバーのアイコンをドラッグ&ドロップでグループに追加
- 設定アプリからファイル選択ダイアログで追加
- 右クリックメニューから「このグループから外す」「別のグループに移動」「純正タスクバーに戻す」を選択
- グループ間のドラッグ&ドロップにも対応

### グループの管理

- 設定アプリから新規グループを作成し、純正タスクバーにピン留め
- グループが空になってからでないと削除できない（誤操作防止）
- グループ名は設定アプリで編集可能

### 設定アプリ

- タスクトレイのStackBarアイコンをクリックして開く
- ユーザーが変更できるデザイン設定
  - テーマ（ダーク・ライト）
  - アイコンサイズ
  - 文字サイズ
  - ポップアップの角の丸さ
  - 背景の透明度
  - 言語（日本語・英語・中国語・ロシア語）

### インストーラー

- セットアップ.exeをダブルクリックでインストール
- PC起動時のスタートアップ自動登録あり
- 先生や友人への配布を想定した本格的な形式

- バリデーションの実装（空のグループ名を弾く）

**今日学んだこと**

- WPFはUIとロジックが分離されている（`.xaml`が見た目、`.xaml.cs`が動き）
- `Grid`の`ColumnDefinitions`と`RowDefinitions`でレイアウトを組む
- `ObservableCollection`を使うとリストの変更がUIに自動反映される
- `DataContext = this`でXAMLからC#のプロパティを参照できる（データバインディング）
- `{Binding Name}`でC#のプロパティをXAMLに繋げる
- `JsonSerializer`でC#のクラスとJSONを相互変換できる
- アプリの設定ファイルは`AppData/Roaming`に保存するのがWindowsの作法
- `string.IsNullOrWhiteSpace()`で空文字・スペースのバリデーションができる
- `ShowDialog()`でダイアログを開き、`DialogResult`で結果を受け取る
- namespaceはプロジェクト内で統一しないとエラーになる
